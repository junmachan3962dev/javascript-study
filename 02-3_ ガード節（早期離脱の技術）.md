#### 1. ガード節の目的

ガード節の最大の目的は、「**コードの『読みやすさ』と『安全性』を劇的に上げること**」です。

具体的には、以下の3つのメリットを得るために使います

 ■**脳の「一時メモリ」を節約する**

通常の `if...else` は、最後まで読まないと「結局いつ終わるのか」が分かりません。ガード節を使うと、**「あ、ここはもう関係ない」と即座に判断して、その条件を頭から消去**できます。

■「**正常な処理」を主役にする**

ガード節を使わないと、一番大事なメイン処理が `if` の深い入れ子（ネスト）の中に隠れてしまいます。 ガード節は、**邪魔な例外をすべて排除して、メイン処理を一番左端（ネストなし）に持ってくる**ためにあります。

■「**前提条件」を保証する**

ガード節を通った後のコードは、いわば「**審査を通過したエリートデータ**」だけが残っている状態です。 「データが空だったらどうしよう？」という不安なしに、のびのびとメイン処理を書くことができます。

#### 結論：何のための技術か？

一言で言えば、「**迷路（ネスト）を壊して、一本道にするため**」の技術です。

「条件に合う時だけ中に入れる（許可制）」から、「条件に合わない奴は追い出す（門番制）」に考え方を切り替えるだけで、コードの複雑さは激減します。

#### 2. ガード節の種類

共通の目的は、「**ダメな条件を先に追い出し、メインの処理を真っ直ぐにする**」ことです。

■ **関数を終わらせる（`return`）**

最も一般的なガード節です。関数の最初で「例外」をすべて追い出します。

```
function deposit(amount) {
  if (amount <= 0) return; // ガード（0以下は即終了）
  
  // 以降は「正しい金額」である前提で書ける
  balance += amount;
}
```

■ **ループをスキップする（`continue`）**

「特定の条件だけ無視したい」ときに使う、ループ内のガード節です。

```
for (const user of users) {
  if (!user.isActive) continue; // ガード（非アクティブなら次へ）

  sendMail(user); // アクティブな人にだけ届く
}
```

■ **論理演算子によるガード（`&&`）**

if文すら使わない、最小のガード節です。

```
// userが存在する「時だけ」、名前を表示する
user && console.log(user.name);
```
