### 1. 鉄則：${ } の中は「式の予約席」

前章の結論通り、**「式（代入できるもの）**」だけがここに入れます。

- **OK（式）**: リテラル、変数、計算、呼び出し式
- **NG（文）**: `if`, `for`, `const`（値を返さないため、埋め込めない）

### 2. 動的評価のプロセス

文字列を作る「直前」に、中の式が評価されます。

1. **評価**: `${ }` の中身を計算して**「値」**にする。
2. **結合**: その値を文字列として合体させる。

### 3.三項演算子との相性

**テンプレートリテラルに埋め込めるので、テンプレートリテラルと三項演算子は最強のタッグ）です。**
```
`あなたは ${age >= 20 ? "大人" : "子供"} です`
```

**三項演算子は現代のJavaScriptに欠かせないので下記に詳しく記したいと思います**

### 4. 三項演算子（Conditional Operator）

三項演算子は条件演算子です（[[01-4_演算子の種類と優先順位]]を参照）

■ **三項演算子とは「式のif文」**

最大の特徴は、`if` が「文」であるのに対し、三項演算子は「**評価されると値を返す『式』**」であるという点です。

■ **基本構造**

3つのパーツで構成されるため「三項」と呼ばれます。
```
条件式 ? 式1（trueの時） : 式2（falseの時）
```

＊この全体が評価されて、「**式1」または「式2」のどちらか一つの「値**」になります。

■ **なぜ重要なのか：代入と埋め込み**

「式」であるからこそ、`if` にはできない以下の芸当が可能です。

**変数に直接代入できる**
```
 const status = (age >= 20) ? "大人" : "子供";
```

■ **三項演算子の実務でよく使われるパターン**

三項演算子が「式（値になる）」である特性を活かし、実務で多用される場面は下記の３つが多いです。

1. **変数へのスマートな代入**

`if` 文を使うと数行かかる処理を、1行で完結させられます。
```
const message = isLoggedIn ? "ようこそ" : "ログインしてください";
```

 ＊変数を `const`（定数）にできるため、後から意図せず値が変わるバグを防げます。


2. **モダンフロントエンド（Reactなど）**

Reactなどのライブラリでは、HTMLのような構造（JSX）の中「**式」しか書けない場所**が多発します。
```
// 画面表示の切り替え
return (
  <div>
    {isLoading ? <Spinner /> : <Content />}
  </div>
);
```

ここでも「値を返す」という式の性質が、画面を動的に切り替える唯一の手段として重宝されます。


3. **関数からの即時リターン**

「条件によって返す値を変えたい」だけのシンプルな関数に最適です。
```
const getFee = (isMember) => isMember ? 500 : 1000;
```


 **まとめ：三項演算子の真価：** 「条件分岐の結果を、そのまま**変数という箱**や**テンプレートリテラルという枠**にパッと放める」という機動性にあります。
