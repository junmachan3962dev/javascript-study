ショートサーキット（短絡評価）※」というJSの性質を利用して、コードを劇的にスリムにする技術。

※結果が決まった瞬間に右側の評価を切り上げる性質。

#### 1. 各演算子の基本とサンプル

まずは、それぞれの演算子が「どう動くか」を整理します。

- **`||`（論理和）**
    
    - **ルール：** 左が **Falsy** なら右を見る。
    - **コード：** `const name = "" || "Guest";` → 結果: `"Guest"`

- **`??`（Null合体演算子）**
    
    - **ルール：** 左が **null / undefined** の時だけ右を見る。
    - **コード：** `const price = 0 ?? 1000;` → 結果: `0` （※`||`だと1000になる）

- **`&&`（論理積）**
    
    - **ルール：** 左が **Truthy** なら右を見る。
    - **コード：** `const result = true && "完了";` → 結果: `"完了"`

- **`?.`（オプショナルチェイニング）**
    
    - **ルール：** 左が **null / undefined** ならそこで処理を止める。
    - **コード：** `const city = user?.address;` → 結果: `user`がいなければ `undefined`

#### 2. 【実戦】よく使われる活用パターン

上記の仕組みを、実際の開発では以下のような「目的」で使い分けます。

これらは代表的な「型」であり、アイデア次第でさらに応用が効きます。

| **活用シーンの例** | **書き方のイメージ**         | **狙い（メリット）**              |
| ----------- | -------------------- | ------------------------- |
| **デフォルト値**  | `const port = input` |                           |
| **数値の保護**   | `input ?? 0`         | 0を「無」と扱わず、数値として活かす        |
| **条件付き実行**  | `isReady && start()` | if文を使わず、条件成立時のみ動かす        |
| **連鎖エラー防止** | `data?.item?.name`   | 深い階層を探索中、途中でデータが無くても落とさない |

#### 3. 実戦リファクタリング例

「紹介」を「技術」に変える書き換え例です。

**Before: if文で愚直に書く**

```
let theme;
if (setting.theme) {
  theme = setting.theme;
} else {
  theme = "light";
}
```

**After: 短絡評価で「小さく、シンプルに」**

```
// 1行で完結。読みやすく、代入ミスも減る
const theme = setting.theme ?? "light";
```
