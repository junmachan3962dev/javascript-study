### 1 オブジェクトの操作（基本）

```
const user = {
  name: "相棒",
  level: 10
};

// 1. 取り出し（ドット記法）
// 基本はこっち！
console.log(user.name); 

// 2. 取り出し（ブラケット記法）
// 変数を使って中身を選びたい時に使う！
const key = "level";
console.log(user[key]); // 10
```

■ **ポイント**

- **ドット（`.`）**: 普段使い用。
- **ブラケット（`[]`）**: どの項目を出すか「後から決めたい」時用。

■**更新と追加**

```
user.level = 11;      // 更新
user.skill = "JS";    // 新しく追加
```
追加と更新はモダンな環境では「元のデータを書き換えない」というルールがあるので好ましくない。
なので元のデータを残したまま新しいデータを作るという方法を使用する
（例）**スプレッド構文を使用する**


### 2. 分割代入

```
const user = {
  name: "相棒",
  level: 10
};

// 1. 同名の変数で取り出す
const { name, level } = user;
console.log(name);  // "相棒"
console.log(level); // 10
```

■ **分割代入の仕組み（分割代入のパターン）**

```
const { name, level } = user;
```
内部的には、`{ name, level }` と書くだけで、JavaScriptが以下の処理をセットで行っています。

1. `const name` という変数を宣言する。
2. `const level` という変数を宣言する。
3. それぞれに `user.name` と `user.level` の値を代入する。

つまり、**「変数の宣言」と「値の抽出」をセットで行うためのショートカット記法**なのです。

＊**ポイント**
- **名前のバインド**: `{ }` の中の名前が、オブジェクトのキーを探しにいってくれる。
- **シンプル化**: `const name = user.name;` と2回書かなくて済むので、「小さく、シンプルに」書ける。

■ **分割代入の応用**

```
const user = { name: "相棒", level: 10 };

// 1. 基本：同名の変数で取り出す
const { name, level } = user;

// 2. 応用：別名を付けて取り出す（コピーして移し替え）
const { name: userName } = user;

// 3. 応用：デフォルト値を設定する（予備の準備）
const { hp = 100 } = user;
```

＊**ポイント**
- 1. **時短**: 変数の宣言と代入を 1 行で済ませるための記法。
- 2. **リネーム**: `:` の右側が「新しく作る変数名」。
- 3. データがなかった時のための予備（デフォルト）です。（hpのデータがあるときはそちらの数値が優先）

### 3. オブジェクトのコピーと更新（スプレッド構文）

■ **概念**

**不変性（Immutable）の維持** 
元のオブジェクト（マスター）を直接書き換えるのではなく、スプレッド構文を使って「**中身が同じ別物**」を新しく作成します。これにより、データの変化を安全に管理できます。

■ **構文**

```
const 新しいオブジェクト = { ...コピー元オブジェクト };
```
- `...`（ドット3つ）は「中身をバラバラに広げる」という命令。

■ **実際のコード**

```
const user = { name: "相棒", level: 10 };

// 1. 全コピーして新しいデータを作る
const copyUser = { ...user };

// 2. コピーしつつ、一部だけ更新する（後から書いた値が優先される）
const updatedUser = { ...user, level: 11 };

// 3. コピーしつつ、新しい項目を追加する
const newUser = { ...user, skill: "JS" };
```

■ **ポイント**

- **非破壊**: 元の `user` は一切変わりません。
- **展開（Spread）**: `{ ...user }` は、`user` という箱を外して、中の具材（nameやlevel）だけを新しい `{ }` に並べ直すイメージで

■ **データの数が多い時の対処**

- **末尾に注目する**: 変更したいキーを必ず一番最後に書くことで、そこが「変更の目印」になる。
- **オブジェクトの分割**: データが多くなりすぎたら、オブジェクト自体を小さく切り分ける（例：`status`、`profile`、`inventory` など）。
