## 1. if文

### 1-1. if文（条件分岐）

条件が **`Truthy`**（真）か **`Falsy`**（偽）かによって、処理を分ける「文」です。

■ **基本形**

```
const score = 85;

if (score >= 80) {
  // 条件が「真」のとき実行
  console.log("合格！");
} else {
  // 条件が「偽」のとき実行
  console.log("不合格...");
}
```

■ **多分岐（else if）**

上から順に判定し、**最初に「真」になったブロックだけ**を実行します。

```
const time = 15;

if (time < 12) {
  console.log("午前");
} else if (time < 18) {
  console.log("午後");
} else {
  console.log("夜");
}
```

### 1-2. 使い分けのコツ：式 vs 文

- **三項演算子（式）**: 判定して **「値」** を選びたいとき。 `const message = score >= 80 ? "合格" : "不合格";`

- **if文（文）**: 判定して **「動作（処理）」** をさせたいとき。 `if (score >= 80) { sendEmail(); }`

### 1-3. 注意：中カッコ `{ }` の省略

処理が1行だけの場合、`{ }` を省略できますが、**実務ではバグ防止のため常に付ける**のが一般的です。

```
// 省略できるが、推奨されない
if (isReady) console.log("OK"); 

// 推奨：短くても { } を付ける
if (isReady) {
  console.log("OK");
}
```

### 14.if文のテクニック（ガード節）

**ガード節（早期リターン）と呼ぶテクニックです。**

「もし〜なら、ここで終了」という条件を先に書くことで、`else` のネスト（入れ子）を深くせずにコードをスッキリできます。

■ **改善前：ネストが深い（読みにくい）**

```
function checkScore(score) {
  if (score >= 0) {
    if (score >= 80) {
      console.log("合格！");
    } else {
      console.log("不合格");
    }
  } else {
    console.log("不正な数値です");
  }
}
```

■ **改善後：ガード節を使用（シンプル！）**

ダメな条件を先にチェックして、すぐに `return`（終了）させます。

```
function checkScore(score) {
  // ガード節：例外を先に弾く
  if (score < 0) {
    console.log("不正な数値です");
    return; // ここで終了
  }

  // 本編の処理
  if (score >= 80) {
    console.log("合格！");
  } else {
    console.log("不合格");
  }
}
```

＊**なぜ「ガード節」を使うのか？**

- **読みやすい**: 「例外処理」と「メイン処理」がハッキリ分かれる。
- **ネストが浅くなる**: 右にコードがズレていかないので、構造がシンプルになる。
 - **コツ：** 「ダメなパターンは最初に追い出す」のが、綺麗な `if` 文のコツです。


## 2. for文（繰り返し）

### 2-1. for文

決まった回数だけ、同じ処理を自動で繰り返すための仕組みです。

■ **基本形**

```
for (初期化式; 条件式; 変化式) {
  // 繰り返したい処理
}
```

サンプルコード
```
// 0からスタートし、3未満の間、1ずつ増やしながら繰り返す
for (let i = 0; i < 3; i++) {
  console.log(i + "回目の実行");
}
```

### 2-2各パーツの役割

1. **初期化式 (`let i = 0`)**: 最初に一度だけ実行。カウンタ変数を作ります。
    
2. **条件式 (`i < 3`)**: ループのたびに判定。**`Falsy`** になったら終了します。
    
3. **変化式 (`i++`)**: 処理が終わるたびに、値を更新します。

■**インクリメントとデクリメント**

変数の値を 1 増やしたり減らしたりする専用の演算子です。

- **インクリメント (`++`)**: 値を **1 増やす**
    `i++` は `i = i + 1` と同じ意味
- **デクリメント (`--`)**: 値を **1 減らす**
    `i--` は `i = i - 1` と同じ意味

### 2-3. for文の制御（break / continue）

ループの動きを途中で変えるための命令です。
#### ■`break`（見つけたら即終了）

効率を上げるため、目的を達成したらループを抜けます。
「検索」や「エラーによる強制終了」でよく使います。

```
// 3を見つけたら、それ以降の数字は見ない
for (let i = 1; i <= 5; i++) {
  if (i === 3) {
    console.log("3を発見！ループを終了します。");
    break; 
  }
  console.log(i + "番目をチェック中...");
}
```

#### ■`continue`（特定の時だけ飛ばす）

不適切なデータを無視して、次の回へ進みます。
「除外リスト」や「スキップ処理」でよく使います。

```
// 偶数の時だけ飛ばして、奇数だけ表示する
for (let i = 1; i <= 5; i++) {
  if (i % 2 === 0) {
    continue; // 以下の処理を飛ばして、次の i へ
  }
  console.log(i + "は奇数です");
}
```
