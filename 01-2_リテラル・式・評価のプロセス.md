## 1. リテラル（Literal）とは

ソースコード上で、**値を直接記述する際の「書き方のルール**」のこと。 JavaScriptエンジンは、この書き方を見て「これは数値だ」「これは文字列だ」と判断します。

- **数値リテラル**: `100`, `3.14`
    　数値として**スタック**へ
- **文字列リテラル**: `"Hello"`, `'World'`
     文字列として**スタック**へ
- **オブジェクトリテラル**: `{ key: "value" }`
    　住所を**スタック**、実体は**ヒープ**へ
- **配列リテラル**: `[1, 2, 3]`
     住所を**スタック**、実体は**ヒープ**へ
- **テンプレートリテラル**:   ` `` `（バッククォート）
 　　**評価してからメモリへ**

## 2. 式（Expression）と評価（Evaluation）

#### 1. 式（Expression）の定義

「式」とは、JavaScriptエンジンに対して「**評価して、一つの値を返せ」と命じるコードの最小単位**です。
＊解析フェーズで式かどうかを判断して、実行フェーズで中身（組み込みオブジェクトか自作の変数なのかの種類）を判断する

> ★**最重要ルール**：
> 「**変数に代入できるもの」は、すべて「式」である。**
#### 2. 式のバリエーションと評価結果

エンジンが「式」をどう評価し、どんな「値」を導き出すかのリストです。

|**式の種類**|**コード例**|**評価された結果（値）**|
|---|---|---|
|**リテラル**|`100`|`100` （そのまま）|
|**変数参照**|`userName`|その変数のメモリに格納されているデータ|
|**演算式**|`10 + 20`|`30` （計算結果）|
|**比較式**|`age >= 20`|`true` または `false`|
|**呼び出し式**|`getName()`|関数の `return` で返される値|

＊**呼び出し式と組み込みオブジェクトは別物なので注意**
組み込みオブジェクト`Math.random()` の例

| **パーツ**         | **正体**          | **役割**                     |
| --------------- | --------------- | -------------------------- |
| **`Math`**      | **組み込みオブジェクト**  | 数学的な機能が詰まった「道具箱」。          |
| **`.random`**   | **メソッド（プロパティ）** | 道具箱の中にある「乱数を作る」という名前の「機能」。 |
| `Math.random()` | **呼び出し式**       | 全体として評価され１つの値を返す           |

#### 3. 「文（Statement）」との決定的違い

ここが「式」を理解する最大の鍵です。

- **式 (Expression)**：値を生成する。代入できる。
    例：`10 + 20`    
- **文 (Statement)**：処理の「枠組み」や「命令」。値を持たない。
    例：`if (...) { ... }` や `for (...) { ... }`
  ＊`const a = if (...)` とは書けません。なぜなら `if` は値を返さない「文」だからです。

**まとめ：変数には式は値を生むので代入できるが文は値を生まないので代入できない**

## 3. 評価のタイミング（フェーズ）

01-1で触れた通り、JavaScriptには2つのフェーズがあります。

1. **評価フェーズ（パース時）**: コードをスキャンし、リテラルが「どんな種類（型）のデータか」を認識し、メモリの準備をする。
    
2. **実行フェーズ（ランタイム）**: 実際に「式」を計算し、確定した「値」をメモリの住所（変数など）に書き込む。

## 4. なぜ「評価」を理解すると強いのか

「式」と「評価」を知っていると、以下のような複雑なコードもシンプルに解釈できるようになります。

```
// 例：条件式の評価
let result = (10 > 5); 
```

1. `(10 > 5)` という**式**が評価される。
2. 評価の結果、`true` という**値**が導き出される。
3. その値が、変数 `result` に代入される。

> **★戦略的視点：** 今後の学習（テンプレートリテラル、条件分岐、関数の戻り値）は、すべて「**式が評価されて値になる**」というこの一点に集約されます。
